# Next Session Prompt

아래 프롬프트를 새 세션에 그대로 붙여 넣으면 된다.

```text
나는 Java 백엔드 개발자 포트폴리오를 강화하기 위해 새 대표 프로젝트를 만들고 싶다.

이전 논의에서 도메인은 "카드 실적 관리 및 결제 카드 추천 서비스"로 정했다.
내 실제 상황은 삼성카드, 국민카드 신용/체크, 현대카드를 쓰고 있고,
매달 카드별 실적을 채워야 하며 우선순위도 상황에 따라 바뀐다.
그래서
- 지금 각 카드 실적이 어디까지 찼는지 보고 싶고
- 특정 금액을 결제할 때 어느 카드로 결제하는 게 좋은지 추천받고 싶다.

이 프로젝트는 Java/Spring 기반으로 만들고 싶고,
헥사고날 아키텍처 + 멀티모듈 Gradle 구조를 설득력 있게 적용하고 싶다.
중요한 건 단순 CRUD가 아니라 도메인 모델과 의사결정 로직이 살아 있는 구조다.

다만 이 세션에서는 바로 구현을 깊게 들어가기보다,
먼저 아래를 순서대로 구체화해 줘.

1. 프로젝트 이름 후보 5~10개와 각각의 톤/장단점
2. 한 줄 소개와 포트폴리오에서 어떻게 포지셔닝할지
3. MVP 범위와 제외 범위
4. 핵심 도메인 모델 초안
5. 헥사고날 + 멀티모듈 구조 제안
6. 초기 패키지/모듈 구조
7. ADR 문서 초안 목록
8. GitHub Projects 보드 구조
9. 첫 2주 개발 계획

주의사항:
- 현재 블로그 저장소에서 작업하지 말고, 새 저장소 기준으로 생각해 줘.
- 필요 이상으로 Kafka, Redis, WebFlux 같은 기술을 억지로 넣지 말아 줘.
- 이 프로젝트는 "설계/도메인 판단/문서화"가 핵심이다.
- 실무 포트폴리오로 설득력 있게 보이도록 판단해 줘.

참고로 이 프로젝트는 나중에 내 GitHub 블로그 포트폴리오의 대표 프로젝트로 올릴 예정이다.
그래서 결과물은 프로젝트 자체뿐 아니라, 나중에 블로그 글과 프로젝트 상세 페이지로 연결하기 쉬운 구조였으면 좋겠다.
```

관련 배경 정리는 `docs/CARD_SERVICE_HANDOFF.md`에 있다.
